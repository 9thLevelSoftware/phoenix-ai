Role: project coordination and API verification implementation.

Task: add no-dependency verification scripts and docs that prove Phase 1 contracts and provide a smoke path for live Cloudflare behavior.

Scope:
- Read targets: phase context, Plan 01-01 summary, Plan 01-02 summary, `package.json`, `src/index.ts`, `wrangler.jsonc`, `README.md`, `worker-configuration.d.ts`.
- Write targets: `package.json`, `tests/phase1-contract.test.mjs`, `scripts/verify-worker-smoke.mjs`, `README.md`, `worker-configuration.d.ts`.
- Forbidden writes: `src/index.ts`, `wrangler.jsonc`, `.dev.vars.example`, `kb/`, `dataset/`, `train_phoenix.py`.

Allowed tools/actions:
- Use read-before-write, evidence-before-action, minimal diff, verify-before-report.
- Use Node built-in modules and Node's built-in `node:test` only.
- Run `npm run typecheck`, `npm run test:contracts`, `node scripts/verify-worker-smoke.mjs --help`, and `git diff --check`.
- Run `npx wrangler types` or the repo-standard Wrangler type generation command only if it is available and does not require deploying or printing secrets.

Forbidden actions:
- Do not add dependencies.
- Do not modify runtime source; Plan 01-02 owns `src/index.ts`.
- Do not perform a live coach request unless the required local secrets and AI Search binding access are present.
- Do not print secrets in test or smoke output.
- Do not skip or weaken tests to make verification pass.

Implementation sequence:
1. Add `test:contracts` and `smoke:worker` scripts to `package.json`, preserving existing scripts.
2. Create `tests/phase1-contract.test.mjs` using `node:test` and `node:assert/strict`. Tests must read source/config files and assert:
   - route literals `/api/health` and `/api/coach` are present
   - `wrangler.jsonc` does not contain `AZURE_OPENAI_API_KEY`
   - `src/index.ts` contains the validation constants and trusted debug helper
   - `src/index.ts` contains `CF_API_TOKEN`, `PHOENIX_DEBUG_TOKEN`, and `cf-aig-gateway-id`
   - `src/index.ts` does not contain the exact raw debug field pattern `systemPrompt: systemPrompt`
   - README contains the canonical secret and verification commands
3. Create `scripts/verify-worker-smoke.mjs` with a `--help` mode, `PHOENIX_BASE_URL` defaulting to `http://127.0.0.1:8787`, and checks for:
   - `GET /api/health` returns 200 JSON with `ok: true`
   - invalid JSON to `/api/coach` returns 400
   - missing `message` returns 400
   - unauthorized `debug: true` returns 403
   - if `PHOENIX_RUN_LIVE_COACH=1`, a bounded live `/api/coach` request returns JSON with `response`; otherwise print that live coach smoke was not requested
4. Update README with exact commands for `npm run test:contracts`, `npm run smoke:worker`, `PHOENIX_RUN_LIVE_COACH=1`, and the prerequisite `npm run dev` or remote `wrangler dev` session.
5. Refresh `worker-configuration.d.ts` using the available Wrangler type generation command. If the command cannot represent `KB`, `CF_API_TOKEN`, or debug env in this repo shape, edit the generated snapshot only if the repo already treats it as checked-in generated output and document the command/result in the summary. If generation is impossible, emit `BLOCKED` with the exact error.
6. Run verification.

Required interfaces/content structure:
- `tests/phase1-contract.test.mjs` must be executable with `node --test tests/phase1-contract.test.mjs`.
- `scripts/verify-worker-smoke.mjs --help` must exit 0 without requiring a running Worker.
- `scripts/verify-worker-smoke.mjs` without `--help` must exit non-zero if the Worker target is unreachable.
- README must explain that live coach smoke needs configured secrets and AI Search access.

Edge/error cases:
- If `node --test` is unavailable in the local Node version, emit `BLOCKED` and report `node --version`.
- If package JSON formatting would be damaged, use `npm pkg set` or a structured JSON edit, then inspect the diff.
- If the live smoke script receives a non-JSON response, print status and a bounded preview only; do not dump long bodies or secrets.
- If the generated Worker env type cannot be refreshed because Wrangler requires authentication or network access, emit `BLOCKED` with the exact command and error.

Verification criteria:
- `npm run typecheck` passes.
- `npm run test:contracts` passes.
- `node scripts/verify-worker-smoke.mjs --help` passes.
- README contains smoke instructions.

Final result format:
- Status: completed or BLOCKED.
- Files changed.
- Verification commands and results.
- Whether live coach smoke was run or why it was not run.
