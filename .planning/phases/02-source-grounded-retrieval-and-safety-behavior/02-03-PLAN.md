   `import { buildSystemPrompt } from './prompts';`
   `import { checkSafetyRedFlags } from './safety';`
   `import { extractChunks, sourceMetadata, formatContext, checkContextGrounding } from './retrieval';`
2. Remove the redundant implementation of these functions and local types (e.g. `normalizeChunk`, `extractChunks`, `sourceLabel`, `sourceMetadata`, `formatContext`, `buildSystemPrompt`, etc.) from src/index.ts to avoid duplication, keeping only the types or schemas needed for interface types.
3. In `handleCoach()`:
   - Perform the safety pre-check immediately:
     `const safetyCheck = checkSafetyRedFlags(body.message);`
     `if (!safetyCheck.safe) { return jsonResponse(env, 200, { response: safetyCheck.responseOverride || '', sources: [] }); }`
   - Execute retrieval as normal. Pass chunks to `sourceMetadata(chunks)` and `formatContext(chunks)` from the imported module.
   - Build system prompt using `buildSystemPrompt`.
   - Complete LLM Gateway fetch.
   - Extract raw response text `replyText`.
   - Validate grounding:
     `const groundingCheck = checkContextGrounding(chunks, replyText);`
     `let finalResponse = replyText;`
     `if (!groundingCheck.grounded) { finalResponse += '\n\n*Note: Some details in this response could not be verified by our official records. Please consult the official Vitruvian documentation for exact specifications.*'; }`
   - Inject the safe `sources` metadata in the success payload:
     `const responsePayload: JsonObject = { response: finalResponse, sources: sourceMetadata(chunks) };`
4. Confirm compile correctness with `npm run typecheck`.
